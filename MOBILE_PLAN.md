# Korneo Mobile: План миграции (Expo/React Native)

> План перехода проекта Korneo на мобильную платформу с сохранением существующего PWA и общим бэкендом Supabase.

## 1. Архитектура Monorepo

Используется структура `yarn workspaces` или `pnpm`:

```text
korneo/
├── apps/
│   ├── web/          # Текущее PWA (React + Vite)
│   └── mobile/       # Новое приложение (Expo SDK 51+)
├── packages/
│   ├── domain/       # Общие типы, константы, бизнес-логика
│   └── api/          # Общий клиент Supabase и API-методы
├── supabase/         # Конфигурация бэкенда (Shared)
└── .github/          # Общие CI/CD воркфлоу
```

## 3. Требования к UI/UX

- **Стиль**: Сохранение эстетики "Матрицы" (Dark mode, зеленые акценты, моноширинные шрифты).
- **Offline First**: Кэширование данных через React Query для работы на объектах с плохой связью.
- **Производительность**: Минимизация ререндеров в списках (FlashList).

## 4. Схема экранов (Sitemap)

- `/auth` — Вход / Регистрация
- `/(app)/index` — Dashboard
- `/(app)/tasks` — Список задач
- `/(app)/tasks/[id]` — Детали задачи
- `/(app)/avr` — Акты выполненных работ
- `/(app)/purchase` — Заявки на закупку
- `/(app)/messages` — Список чатов
- `/(app)/messages/[id]` — Окно чата
- `/(app)/profile` — Настройки профиля
- `/(app)/map` — Карта объектов

## 5. CI/CD и Деплой

- **GitHub Actions**: Автоматическая сборка `.apk` и `.ipa` через EAS Build.
- **EAS Update**: Доставка критических обновлений JS-бандла без переустановки приложения.
- **Sentry**: Отслеживание ошибок на мобильных устройствах.
